## 一、项目全景

- **做了什么**:清掉旧工具 Gridea → 用 **Astro** 从零搭一个静态博客「心智边界」(德语 / 小说 / AI 三板块)→ 按 Claude Design 的设计稿重构 UI → 部署到 **GitHub Pages** 上线 → 发布第一篇真文章。
- **最终技术栈**:Astro(SSG)+ Markdown 内容集合 + 原生 CSS + Git/GitHub + GitHub Actions 自动部署。

## 二、技术要点(掌握 / 接触到的概念)

| 概念 / 技术 | 一句话定位 |
|---|---|
| **静态站点生成 SSG** | 打包时把页面预先生成成 HTML,访客直接拿成品,快且稳(全项目的根本范式) |
| **Astro 框架** | 一个 SSG 框架;结构 = `src/pages`(页面)`layouts`(外壳)`components`(积木)`content`(内容)`styles`(样式)+ `astro.config`、`node_modules`、`public` |
| **组件 Component / 布局 Layout** | 把界面拆成可复用单元;Layout 是页面模板 |
| **插槽 Slot** | 模板里的"占位洞",每页把自己内容塞进去 |
| **属性 Props** | 给组件传参数,让同一组件产出不同结果(参数化) |
| **数据驱动渲染(数组 + `.map`)** | 界面 = 数据 + 模板;改数据即改界面(可扩展的根) |
| **内容集合 Content Collections** | `content.config.ts` 用 **zod schema** 定字段规范 + **glob loader** 收集 `.md`;`getCollection` 取数据 |
| **动态路由 `[slug]` + `getStaticPaths`** | 一个模板批量生成所有文章页 |
| **Markdown + Frontmatter(YAML)** | 正文 Markdown;开头 `---...---` 里写 title/date/summary 等元数据 |
| **CSS 设计系统** | CSS 变量(设计令牌)、Flexbox/Grid 布局、`@media` 响应式、伪元素 `::before` 做装饰(如藤蔓节点) |
| **Git / GitHub** | 版本控制:commit 存档、`.gitignore`、分支、远程仓库 |
| **GitHub Pages + GitHub Actions** | 免费托管 + CI/CD:`push` 触发 `.github/workflows/deploy.yml` 自动构建发布 |
| **gh CLI / 包管理(Node/npm)** | 命令行操作 GitHub;`package.json` 声明依赖与命令 |
| **Claude Design / DesignSync(MCP)** | 把设计稿源码(`.dc.html`)读进项目,翻译成组件 + CSS |

## 三、技术难点(真正踩过的坑 + 解法)

| # | 难点 / 坑 | 解决办法 |
|---|---|---|
| 1 | **Frontmatter 没闭合** → 构建报错(出现过两次) | 开头信息块必须有结尾 `---`,正文写在它下面 |
| 2 | **日期要手打、格式易错**(`2026-6-21` 非标准 YAML 日期) | schema 用 `z.coerce.date().default(()=>new Date())` → 可省略、自动用今天;并配 VS Code 模板 |
| 3 | **DesignSync `get_file` 有 256KB 上限** → 打包版 `.html` 被截断、无法还原 | 改读未打包的 **`.dc.html` 源码**(返回里 `truncated:false`) |
| 4 | **打包页格式** = `<script>` 里 base64+gzip 的 manifest/template | 看懂后可用 Node 解码(后发现 `.dc.html` 更直接,绕开) |
| 5 | **DesignSync 需 `/design-login` 授权**(桌面 App 不暴露该命令) | 凭证存于 `~/.claude`,命令行授权一次后通用 |
| 6 | **GitHub 账号不一致**:旧站 `Justin-zyc`,实际登录 `justinzyc271828-sys` | 用 `gh api user` 确认真实账号,再定仓库 / 网址 |
| 7 | **Actions 默认 Node 20,Astro 6 要 ≥22.12** → 首次构建失败 | 流水线里指定 `node-version: 22` |
| 8 | **GitHub Pages 要先开"用 Actions 构建"** → 否则部署步骤失败 | `gh api` 设 `build_type=workflow`,再重跑 |
| 9 | **文件名带 `#`/中文、目录混入** → 构建 `EISDIR` / 缺 title | 删掉残件,用规范英文 slug 的 `.md` |
| 10 | **Windows 换行 CRLF/LF 警告** | 无害,可加 `.gitattributes` 消除(可选) |
| 11 | **暴露过的 GitHub token**(旧 Gridea 配置) | 提醒撤销;新部署改用 `gh` 全新授权 |
| 12 | **本地预览服务器的开 / 关管理** | 改文件自动热重载;空闲时停掉,需要时再起 |

## 四、完整技术步骤(可复现流程)

### 阶段 0 · 清场与环境
1. 关进程、删旧工具的数据 / 程序文件夹 + 开始菜单快捷方式 + 注册表卸载项(Windows)。
2. 确认工具链:Node ≥22、npm、git、VS Code 就位。

### 阶段 1 · 搭骨架
3. `npm create astro@latest`(minimal 模板)生成项目 → 装依赖 → `git init`。
4. 认识项目结构;`npm run dev` 起本地预览(`localhost:4321`)。

### 阶段 2–3 · 外壳 + 内容系统
5. 写 `BaseLayout`(外壳 + `<slot/>`)、`Header` / `Footer`(组件)、`global.css`。
6. `content.config.ts` 定义集合(zod schema + glob loader);`src/content/{de,fiction,ai}/*.md` 放文章。
7. 列表页用 `getCollection` + `.map`;文章页用 `[slug].astro` + `getStaticPaths` + `render()`。

### 阶段 4 · 设计接入与重构
8. `DesignSync`(授权后)`list_files` / `get_file` 读 `.dc.html` 设计源码。
9. 把设计稿翻译成:CSS 设计令牌(变量)+ 组件(Header / Footer / Subscribe / SectionArchive / ArticleLayout)+ 响应式 `@media`。
10. `npm run build` 全量构建,验证零报错。

### 阶段 5 · 上线
11. `astro.config` 设 `site`;写 `.github/workflows/deploy.yml`(指定 Node 22)。
12. `gh auth login` 登录 → `gh repo create <user>.github.io` → `git push`。
13. `gh api ... pages build_type=workflow` 开启 Pages → Action 自动构建发布 → 验证公网 URL。

### 日常发布(已自动化)
14. 写 `.md`(`post` 模板自动填日期)→ `git commit` + `git push` → Actions 自动重新部署,2–3 分钟上线。
