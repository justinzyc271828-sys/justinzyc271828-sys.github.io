## 🕳️ 踩过的坑(别再踩)

**坑 1 · 256KB 截断**
`get_file` 有 **256 KiB 上限**。`心智边界-*.html` 是 base64+gzip 的"打包页",每个都 >256KB,会被截到 262144 字节 → manifest 残缺、template 丢失 → **根本无法解码**。

**坑 2 · 硬解打包版是死路**
试图从截断的残包里抠源码(写宽容 JSON 解析器、逐条解 base64+gzip)纯属白费力:
第一个 54KB 条目是**共享 React 运行时**(三页字节完全相同,SHA 一致),页面专属代码排在它和 10 个字体之后,**正好落在截断点之外**,拿不到。

**坑 3 · 把真源码当垃圾跳过了(最关键)**
文件清单里的 `.dc.html`(自然科技 / 方向对比)一开始被误当成"无关旧稿"跳过——**它们才是没打包的纯源码本体**。
> 💡 教训:**撞到硬墙时,先回头把手里已有的输入重看一遍、找有没有更软的入口,再决定要不要往硬里钻。** 不要"锁定一个目标文件名 + 一条道走到黑"。

**本地副本 & 清理情况(已就绪 ✅)**
- `.design-ref/home.dc.html`、`list.dc.html`、`read.dc.html` = 三页的**干净源码本地副本**(取自 DesignSync 原文,可直接读)。
- 仍以 DesignSync 实时 `get_file` 的 `.dc.html` 为**权威来源**,本地副本只是便利。
- 已删除:截断打包版 `home.html / list.html / read.html`,以及旧脚本 `decode-bundle.mjs / extract-designs.mjs / inspect-bundle.mjs`。
- ⚠️ `.dc.html` 依赖 `support.js`(Claude Design 自定义框架)才能在浏览器完整渲染;本地直接双击打开可能不完整,**当源码参考读即可**。

## 🛠️ 落地到 Astro 的注意点

- `.dc.html` 是单文件、**全内联样式** + 自定义 `<x-dc>` / `DCLogic` 框架 + `style-hover` 自定义属性。落地时:内联样式抽成组件 / CSS(或先内联跑通再重构),两段交互用**原生 JS** 重写。
